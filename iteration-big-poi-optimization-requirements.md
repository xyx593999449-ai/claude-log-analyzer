# 大 POI 可视化二期（效果优化）需求文档

## 1. 背景与问题
- 当前首页可用，但视觉风格与原项目差异较大。
- 原始页面（时间线、工具分析、Token 分析、原始日志）不可见。
- 上传区交互弱化，缺少批量选择和拖拽。
- 成本展示口径需统一到 GLM 价格并增加说明。
- 详情页信息密度和视觉层级不足。

## 2. 根因确认（本次先分析不改代码）
- 首页入口被直接切换为 `DashboardHome`，导致原页面导航入口消失。
- 原页面组件文件被移除，无法恢复到原交互和样式。
- 当前上传区按“人工区分文件类型”设计，缺少智能识别和批量导入体验。

## 3. 本次优化目标
- 恢复“原项目页面能力”的可访问性，同时保留新首页看板。
- 上传支持批量、拖拽、自动识别日志类型（核实/质检、执行/Claude）。
- 成本统一按 GLM 单价计算并显式标注公式。
- Token 指标同时展示：总量、输入、输出。
- 重构详情区视觉与信息层级，提升可读性和观感。

## 4. 范围
### 4.1 页面与导航
- 新首页继续作为默认页。
- 新增顶部导航/页签：`看板首页`、`日志分析（原页面）`、`任务日志详情`。
- 恢复原页面入口，支持来回切换。

### 4.2 上传交互
- 支持批量文件选择（`multiple`）。
- 支持拖拽上传（拖入区域 + hover 态）。
- 上传后自动识别分类：
  - 维度一：核实 / 质检
  - 维度二：执行日志 / Claude 日志
- 识别结果可人工修正后再导入。
- 保留“清除缓存（仅临时分析表）”能力。

### 4.3 成本与 Token 展示
- 成本统一公式：
  - 输入成本 = `input_tokens / 1_000_000 * 4`
  - 输出成本 = `output_tokens / 1_000_000 * 18`
  - 总成本 = 输入成本 + 输出成本
- 所有概要卡片和详情行统一采用该口径。
- 在成本字段增加悬浮说明（tooltip）：展示单价和公式。
- Token 展示统一为：
  - 总 Token
  - 输入 Token
  - 输出 Token

### 4.4 详情展示策略
- 列表按“信息分组”展示：POI 基本信息 / 核实执行 / 质检执行 / 结果摘要。
- 长文本采用“摘要 + 展开”。
- 状态异常用统一视觉标识（警示图标 + 悬浮说明）。
- 提升视觉层级：卡片标题、标签色、对齐规则、留白一致性。

## 5. 非功能要求
- 延续现有技术栈（React + Tailwind + Express）。
- 保持大日志场景：导入后仍通过落表查询展示。
- 不修改业务表数据（`poi_init`、`poi_verified`、`poi_qc` 只读）。

## 6. 约束与风险
- 当前存在文本编码显示异常，需避免继续引入乱码文本资源。
- 自动识别日志类型需容错（文件命名不规范、日志内容混合）。
- 恢复原页面能力时需避免与新看板状态管理冲突。

## 7. 本轮补充需求（2026-03-18）
- 任务列表中点击“日志详情”后，展示样式与交互需复用原项目日志详情方案（原时间线/分析视图风格），不再使用当前简版双文本框页面。
- 新日志详情页需支持：
  - 多视图切换（时间线、工具调用、Token、原始日志）；
  - 按任务过滤后的会话数据展示；
  - 与首页风格一致的导航返回。

## 开发进展更新（2026-03-18）

### 已完成
- 首页恢复为看板，并补充导航入口，可切换到“日志分析页”。
- 上传区改为双窗口（核实/质检），每个窗口支持批量选择、拖拽上传、自动识别日志类型（执行日志/Claude 日志），并支持手动覆盖识别结果。
- 成本计算统一按 GLM 定价口径展示：输入 4 元/百万 Token，输出 18 元/百万 Token；页面增加悬浮说明。
- Token 展示调整为“总 Token + 输入 Token + 输出 Token”，覆盖概要卡片与任务详情执行摘要。
- 日志详情页改为复用原项目风格：时间线、工具统计、原始日志三个视图，支持任务级核实/质检日志并行查看。
- 页面文案与样式已统一清理，去除异常乱码文案（终端显示乱码不影响 UTF-8 文件内容）。
