# 大 POI 可视化改造验收方案

## 1. 验收目标

确认本次迭代完成后，页面能够基于 mock 数据库和日志分析结果，稳定展示大 POI 核实与质检任务的概要与详情，并满足大日志场景下的可用性要求。

## 2. 验收范围

- mock 数据库可用
- 日志导入可用
- 临时分析表可用
- 清缓存可用
- 概要展示可用
- 详情展示可用
- 筛选与展开可用
- 异常态与空态可用
- 日志追溯入口可用

## 3. 验收清单

### 3.1 数据层验收

- 能从样例数据初始化 `poi_init`
- 能从样例数据初始化 `poi_verified`
- 能从样例数据初始化 `poi_qc`
- 能创建临时分析表
- 能记录最近一次导入
- 业务表保持只读，不发生修改或删除

### 3.2 日志导入验收

- 能导入核实日志样例
- 能导入质检日志样例
- 导入后能够写入任务维度聚合结果
- 默认追加导入
- 重复导入后以最新结果为准
- 能清除日志落表缓存
- 清缓存后业务基础数据仍完整可读

### 3.3 概要模块验收

#### 模块 1：所有任务

- 能展示 `poi_init` 总量
- 能展示各 `verify_status` 数量

#### 模块 2：已核实任务

- 能展示核实任务量
- 能展示核实总用时
- 能展示平均单条用时
- 能展示平均输入 Token
- 能展示平均输出 Token
- 能展示平均成本

#### 模块 3：已质检任务

- 能展示质检任务量
- 能展示质检总用时
- 能展示平均单条用时
- 能展示平均输入 Token
- 能展示平均输出 Token
- 能展示平均成本
- 质检主状态展示以 `qc_status` 为准

#### 模块 4：人工任务监控

- 能展示需人工核实任务数量
- 能展示需人工核实任务摘要
- 人工任务判定符合以下任一条件：
  - `poi_qc.is_manual_required = 1`
  - `poi_verified.verify_status = "需人工核实"`
- 能展示最近一次导入快照

### 3.4 详情列表验收

- 每个任务一行记录
- 能展示 `poi_init` 基本信息
- 能展示核实成果
- 能展示核实执行详情
- 能展示质检成果与质检详情
- 长文本支持展开 / 收起
- 日志与数据库结果不一致时有明显告警标记和说明
- 可跳转查看该任务的日志详情

### 3.5 筛选与搜索验收

- 可按任务关键字搜索
- 可按核实状态筛选
- 可按质检状态筛选
- 可按是否需人工核实筛选
- 可按异常任务筛选
- 筛选后结果与概要逻辑一致

## 4. 数据正确性抽检

从样例数据中随机抽取若干任务，逐项核对：

- `task_id` 是否正确关联三张业务表和两类日志
- 缺失 `task_id` 的日志是否能基于上下文正确补关联
- `poi_verified` 的结果是否正确映射到页面
- `poi_qc` 的结果是否正确映射到页面
- 开始时间、结束时间、用时是否正确
- 重试次数是否与日志一致
- 输入 / 输出 Token 与 Claude 日志统计是否一致
- 成本是否与日志统计一致
- 状态不一致时是否按数据库结果展示并给出告警原因

## 5. 性能验收

- 导入后页面查询只读数据库，不重新全量解析日志
- 首屏打开时间可接受
- 展开单行详情不应阻塞整个列表
- 大文本字段不会撑坏页面布局
- 首页未导入日志时仍可展示基础任务数据

## 6. 体验验收

- 页面风格与原项目一致
- 状态标签清晰
- 空态文案明确
- 导入中状态明确
- 异常任务有可见提示
- 核实日志和质检日志上传入口分区清晰
- 首页与日志详情页切换路径清晰

## 7. 验收通过标准

满足以下全部条件即可通过：

- 样例数据可正常初始化
- 样例日志可正常导入
- 4 个概要模块展示完整
- 任务列表与详情可正常查看
- 抽检数据无明显错位
- 大日志场景下页面查询基于临时表
- 清缓存不影响业务基础表
- 无阻断性的 UI 问题

## 8. 验收后续动作

验收通过后再进入：

- 正式数据库接入替换
- README / CHANGELOG 最终维护
- 本轮功能收尾说明整理
