面向大 POI 核实与质检场景的日志分析与流程看板。

项目支持日志导入、指标总览、任务流程列表和单任务日志详情分析，适合用于核实链路与质检链路的日常巡检、异常定位和结果复核。

## 1. 核心能力

- 日志导入：支持核实日志与质检日志上传，并触发结果刷新。
- 首页看板：集中展示任务总量、自动化率、核实质量、异常任务等核心指标。
- 流程列表：按任务展示流程进度、核实模块、质检模块、异常风险与元信息。
- 日志详情：支持核实 / 质检阶段切换，查看时间线、工具调用、Token 成本与原始日志。

## 2. 当前界面结构

### 2.1 首页

- 顶部为日志导入区，默认收起。
- 顶部主看板改为全量流程执行状态展示，按流程节点查看整体推进情况。
- 顶部主看板支持地铁线路式流程带展示，主链路按 5 个流程节点平铺呈现整体推进状态。
- 全量流程看板下方展示 3 个核心指标：核实自动化率、核实质量、异常任务。
- 全量流程看板去掉影响观感的装饰虚线，优先保证流程节点本身清晰可读。
- 核实与质检执行概览同一行横向对比。
- 首页保留任务总量、执行概览、流程分布和异常标签等核心信息。
- 当前页流程分布使用整行平铺的任务流转状态板展示，每个状态直接显示任务量和占比。
- 当前页异常信息放在流程分布下方，使用 tag 聚合展示。
- 任务详情使用流程列表卡片展示。

### 2.2 任务详情卡片

- 默认展示任务摘要、地址、证据摘要和异常标签。
- 证据摘要读取 `poi_verified.evidence_record`，展示证据总量与各 `source_name` 的条数。
- 点击“展开更多信息”后展示流程轨道、核实模块、质检模块、异常与风险和日志详情入口。
- 核实模块与质检模块支持共享展开按钮，便于同时查看摘要、时间戳、评分和执行明细。
- 当前流程节点使用靛蓝色高亮。
- 执行异常、需人工介入、质检不通过等状态会重点高亮。

### 2.3 日志详情页

- 使用与首页一致的卡片化风格。
- 顶部展示任务摘要和阶段切换。
- 中部展示单阶段日志分析结果。
- 滚动区域使用接近 IDE / VS Code 风格的滚动条。
- 执行异常会单独以告警卡形式展示。
- 文件上传导入模式会同时保存原始日志内容，供任务日志详情页按批次回放与筛选。


## 5. 指标口径

- 自动化率：`1 - (需人工核实任务数 / 已有核实结论任务数)`
- 核实质量：`(核实通过且质检通过 + 需人工核实且质检不通过) / 已质检总量`
- 成本口径：
  - 输入 Token：`4 元 / 百万 Token`
  - 输出 Token：`18 元 / 百万 Token`
  - 平均成本：`总成本 / 任务数`
