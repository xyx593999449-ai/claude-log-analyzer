# 大 POI 可视化二期（效果优化）验收方案

## 1. 验收环境
- 本地开发环境：前端 `3000`，后端 `3001`
- 使用样例日志 + 实际日志各一组

## 2. 功能验收项
### 2.1 页面恢复与风格一致性
- 打开首页默认进入看板。
- 可切换到“原始日志分析页面”。
- 新旧页面切换后状态与数据不串页、不丢失。

### 2.2 上传能力
- 支持一次选择多个文件。
- 支持拖拽多个文件进入上传区。
- 系统可自动识别文件类型（核实/质检、执行/Claude）。
- 自动识别错误时可手工调整并继续导入。

### 2.3 成本口径与提示
- 成本计算符合 GLM 口径：
  - 输入 4 元 / 百万 token
  - 输出 18 元 / 百万 token
- 悬浮提示可看到口径说明与公式。
- 概要与详情中的成本总额一致。

### 2.4 Token 展示
- 概要区可看到：总 Token、输入 Token、输出 Token。
- 详情区可看到单任务同样三项。
- 汇总逻辑与后端返回值一致。

### 2.5 详情视觉
- 信息分组明确，可快速识别状态、结果、耗时、成本、重试。
- 长文本可展开/收起，不出现布局挤压。
- 异常状态有统一警示样式与说明。

## 3. 回归验收项
- 清除缓存仅影响临时分析表，不影响业务表。
- 筛选、分页、页大小设置功能保持可用。
- 导入追加策略保持不变。

## 3.1 日志详情复用验收（新增）
- 点击任务“日志详情”进入后，页面包含原项目日志分析核心视图：
  - 时间线视图
  - 工具调用视图
  - Token 分析视图
  - 原始日志视图
- 日志详情页展示内容为“当前 task_id 关联会话”，不是全量日志。
- 详情页样式与原项目风格保持一致（导航、间距、卡片层级、交互方式）。

## 4. 判定标准
- P0：上传可用、数据可见、成本口径正确、页面可切换。
- P1：详情样式优化与交互完整。
- 全部 P0 通过且无阻断性回归，判定本次迭代通过。
